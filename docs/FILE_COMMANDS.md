# Команды работы с файлами

## Обзор

Ассистент разработчика поддерживает команды для работы с файлами проекта через MCP (Model Context Protocol). Все команды начинаются с `/` и выполняются в чате с ассистентом.

## 📁 Доступные команды

### Чтение файлов
```
/read <путь_к_файлу>
```
**Описание:** Чтение содержимого файла проекта.

**Примеры:**
- `/read app/main.py` - прочитать главный файл приложения
- `/read README.md` - прочитать документацию проекта
- `/read docs/API.md` - прочитать API документацию

**Ограничения:**
- Максимальное количество строк: 100 (по умолчанию)
- Файл должен находиться внутри проекта
- Поддерживаются текстовые файлы (.py, .md, .txt, .json и др.)

### Поиск по файлам
```
/find "текст для поиска"
```
**Описание:** Поиск текста в файлах проекта.

**Примеры:**
- `/find "class OllamaClient"` - найти определение класса
- `/find "def search_in_files"` - найти функцию
- `/find "import fastapi"` - найти импорты FastAPI

**Особенности:**
- Поиск нечувствителен к регистру
- Ищет во всех файлах проекта (по умолчанию *.py)
- Показывает до 10 файлов с результатами
- Для каждого файла показывает до 3 строк с совпадениями

### Поиск импортов
```
/imports <имя_модуля>
```
**Описание:** Поиск всех мест, где импортируется указанный модуль.

**Примеры:**
- `/imports fastapi` - найти импорты FastAPI
- `/imports httpx` - найти импорты HTTP клиента
- `/imports os` - найти импорты модуля OS

**Особенности:**
- Ищет все виды импортов (`import module`, `from module import`, `import module as alias`)
- Показывает файлы и строки с импортами
- Поддерживает поиск по частичным именам

### Список файлов
```
/files [шаблон]
```
**Описание:** Получение списка файлов по шаблону.

**Примеры:**
- `/files` - все файлы проекта
- `/files *.py` - все Python файлы
- `/files *.md` - все Markdown файлы
- `/files app/*.py` - Python файлы в папке app

**Особенности:**
- Поддерживает glob шаблоны (`*`, `?`, `[]`)
- Максимальная глубина поиска: 3 уровня
- Исключает системные директории (.git, __pycache__, venv и др.)

### Анализ файла
```
/analyze <путь_к_файлу>
```
**Описание:** Анализ структуры файла.

**Примеры:**
- `/analyze app/ollama_client.py` - анализ клиента Ollama
- `/analyze app/file_mcp.py` - анализ инструментов работы с файлами
- `/analyze README.md` - анализ документации

**Анализируемые аспекты:**
- Для Python файлов:
  - Импорты
  - Функции
  - Классы
  - Количество строк
  - Размер файла
- Для Markdown файлов:
  - Заголовки
  - Уровни заголовков
  - Количество строк

### Создание файлов
```
/create <путь_к_файлу> "содержимое файла"
```
**Описание:** Создание нового файла с содержимым.

**Примеры:**
- `/create docs/new_feature.md "# Новая функция\n\nОписание новой функции."`
- `/create test_file.txt "Тестовое содержимое"`
- `/create app/utils/helper.py "def helper_function():\n    return 'Hello'"`

**Особенности:**
- Создает директории, если они не существуют
- Если файл уже существует, создает бэкап
- Поддерживает многострочное содержимое (используйте \n для переносов)

### Добавление в файлы
```
/append <путь_к_файлу> "текст для добавления"
```
**Описание:** Добавление текста в конец существующего файла.

**Примеры:**
- `/append README.md "## Новый раздел\n\nОписание нового раздела."`
- `/append app/main.py "\n# Новый комментарий в конце файла"`
- `/append docs/CHANGELOG.md "\n## Версия 1.1.0\n- Добавлена новая функциональность"`

**Особенности:**
- Если файл не существует, создает его
- При изменении существующего файла создает бэкап
- Добавляет текст в конец файла

## 🔒 Система безопасности

### Проверка путей
Все операции с файлами проверяются на:
1. **Нахождение внутри проекта** - запрещены пути вне проекта
2. **Безопасность символов** - фильтрация опасных символов
3. **Исключение системных директорий** - .git, __pycache__, venv и др.

### Бэкапы
При изменении файлов автоматически создаются бэкапы:
- **Директория бэкапов:** `.file_mcp_backups/`
- **Формат имен:** `filename.backup_YYYYMMDD_HHMMSS`
- **Сохранение:** копия файла перед изменением

### Ограничения
- **Глубина поиска:** максимум 5 уровней для поиска, 3 уровня для списка
- **Количество результатов:** максимум 20 для поиска, 30 для списка
- **Размер чтения:** максимум 100 строк по умолчанию

## 🚨 Ошибки и их решение

### Файл не найден
```
Ошибка: "Файл не существует: app/nonexistent.py"
Решение: Проверьте правильность пути и существование файла
```

### Путь вне проекта
```
Ошибка: "Путь находится вне проекта: /etc/passwd"
Решение: Используйте только файлы внутри проекта
```

### Ошибка доступа
```
Ошибка: "Ошибка чтения файла: [Errno 13] Permission denied"
Решение: Проверьте права доступа к файлу
```

### Слишком много результатов
```
Сообщение: "... и еще 20 файлов"
Решение: Уточните запрос для более точного поиска
```
