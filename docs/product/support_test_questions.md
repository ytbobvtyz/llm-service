# Тестовые вопросы для проверки работы поддержки

## Команды поддержки для тестирования

### 1. Основные команды
```
/support Почему не работает авторизация?
/support Как обновить тарифный план?
/support Какие системные требования для приложения?
/support Как восстановить доступ к аккаунту?
/support Как очистить кэш приложения?
```

### 2. Команды управления
```
/faq
/users
/tickets
/stats
```

### 3. Команды разработчика
```
/help
/branch
/files
/structure
/readme
```

## Примеры естественных вопросов для тестирования

### Вопросы по авторизации
- "Не могу войти в аккаунт, что делать?"
- "Забыл пароль, как восстановить?"
- "Почему выдает ошибку при входе?"
- "Как сменить email для входа?"

### Вопросы по тарифам
- "Хочу перейти на бизнес-тариф"
- "Как отменить подписку?"
- "Какие есть тарифные планы?"
- "Есть ли пробный период?"

### Вопросы по функционалу
- "Как создать новый проект?"
- "Где найти историю заказов?"
- "Как экспортировать данные?"
- "Как настроить уведомления?"

### Технические вопросы
- "Приложение тормозит, что делать?"
- "Не загружаются файлы"
- "Ошибка 500 при сохранении"
- "Как обновить приложение?"

## Ожидаемые ответы

### Для команды `/support [вопрос]`
- Должен вернуть структурированный ответ с решением проблемы
- Должен использовать markdown форматирование
- Должен предлагать контакты поддержки если решение сложное

### Для команды `/faq`
- Должен показать список часто задаваемых вопросов
- Должен быть отформатирован как список

### Для команды `/users`
- Должен показать список пользователей с их тарифными планами

### Для команды `/tickets`
- Должен показать список тикетов с их статусами

### Для команды `/stats`
- Должен показать статистику поддержки (пользователи, тикеты, время ответа)

## Критерии успешного тестирования

1. **Все команды работают** - возвращают ответы без ошибок
2. **Форматирование корректное** - ответы читаемы и структурированы
3. **Контекст правильный** - ответы соответствуют запросам
4. **Производительность** - время ответа < 5 секунд
5. **Интеграция** - веб-интерфейс корректно отображает ответы

## Быстрые тесты через веб-интерфейс

1. Откройте веб-интерфейс по адресу http://localhost:8000
2. Используйте кнопки быстрых вопросов для тестирования
3. Проверьте работу всех команд поддержки
4. Убедитесь, что статус поддержки отображается корректно
