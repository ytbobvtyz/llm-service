# Архитектура проекта

## Общая схема
```
┌──────────────────────────────────────────────────┐
│                 Веб-интерфейс                    │
│           (HTML/CSS/JS - index.html)             │
└─────────────────┬────────────────────────────────┘
                  │ HTTP/WebSocket
                  ▼
┌──────────────────────────────────────────────────┐
│                FastAPI приложение                 │
│                  (app/main.py)                    │
└─────┬──────────────────────┬──────────────────┬──┘
      │                      │                  │
      ▼                      ▼                  ▼
┌────────────┐       ┌──────────────┐  ┌─────────────┐
│  RAG-слой  │       │ MCP Git      │  │ Ollama      │
│  (rag.py)  │       │ (mcp_tools.py) │ │(ollama_client)│
└─────┬──────┘       └──────────────┘  └─────────────┘
      │                      │                  │
      ▼                      ▼                  ▼
┌────────────┐       ┌──────────────┐  ┌─────────────┐
│ Проект     │       │ Git repo     │  │ Ollama API  │
│ Документы  │       │              │  │             │
└────────────┘       └──────────────┘  └─────────────┘
```

## Компоненты

### 1. Веб-интерфейс (`app/static/`)
- `index.html` - основная страница с чатом
- `style.css` - стили
- `script.js` - клиентская логика

### 2. FastAPI сервер (`app/main.py`)
- Точка входа приложения
- Настройка middleware и CORS
- Инициализация компонентов

### 3. API эндпоинты (`app/api.py`)
- Обработка HTTP запросов
- Эндпоинты для чата, RAG и Git операций
- Валидация данных

### 4. Модели данных (`app/models.py`)
- Pydantic модели для запросов/ответов
- Структуры для чата, RAG, Git операций

### 5. Конфигурация (`app/config.py`)
- Настройки приложения
- Константы и переменные окружения

### 6. RAG система (`app/rag.py`)
- **ProjectDocsRAG** - для документации проекта
- Загрузка и индексация документов из папки `docs/`
- Поиск по документации с простым совпадением ключевых слов

### 7. MCP Git инструменты (`app/mcp_tools.py`)
- **GitMCP** класс для работы с Git
- Получение информации о ветке, файлах, структуре проекта
- Интеграция через subprocess и os.walk

### 8. Ollama клиент (`app/ollama_client.py`)
- **OllamaClient** для взаимодействия с Ollama API
- Обработка специальных команд
- Сбор контекста из RAG и Git информации

## Поток данных

### Чат с ассистентом:
1. Пользователь отправляет сообщение через веб-интерфейс
2. Если сообщение начинается с `/`, проверяются команды
3. Если обычное сообщение:
   - Поиск в RAG по документации проекта
   - Формирование промпта с историей и контекстом
   - Запрос к Ollama API
   - Формирование ответа с источниками

### Обработка команд:
1. Распознание команды (начинается с `/`)
2. Вызов соответствующего метода GitMCP
3. Форматирование ответа для пользователя

### Работа с Git через MCP:
1. Вызов subprocess команд (git branch, git diff)
2. Обход файловой системы для структуры проекта
3. Возвращение результатов в структурированном формате

## Базы данных

### `data/project_docs.db` (планируется)
- Векторная база для документации проекта
- Для эффективного семантического поиска

### `data/metadata.db`
- Существующая база для документов логистики
- Используется FAISS для векторного поиска

## Взаимодействие с внешними сервисами

### Ollama
- Модель LLM для генерации ответов
- REST API для выполнения запросов
- Поддержка настроек температуры и токенов

### Git
- Локальный репозиторий проекта
- Использование командной строки через subprocess
- Чтение состояния репозитория без записи

## Безопасность
- CORS настройки для веб-интерфейса
- Валидация входных данных через Pydantic
- Обработка ошибок в API эндпоинтах
- Безопасное использование subprocess

## Расширяемость

### Добавление новых команд:
1. Добавить обработку в `_handle_command()` в `app/ollama_client.py`
2. Добавить соответствующую функцию в `app/mcp_tools.py`
3. Обновить `/help` команду

### Добавление новых источников документации:
1. Добавить файлы в папку `docs/`
2. Будет автоматически загружено в ProjectDocsRAG при запуске

### Интеграция с другими MCP серверами:
1. Создать новые классы аналогично GitMCP
2. Добавить соответствующие эндпоинты в `app/api.py`
3. Обновить обработку команд
