# LLM Service - Ассистент разработчика

## День 31+: Ассистент разработчика с RAG, MCP и работой с файлами

Этот проект реализует ассистента разработчика с возможностью доступа к документации проекта через RAG, к Git репозиторию через MCP (Model Context Protocol) и работы с файлами проекта.

## 🚀 Функциональность

### 📚 RAG для документации проекта
- Автоматическая загрузка документации из папки `docs/`
- Быстрый поиск по документации проекта
- Интеграция с существующим RAG для документов логистики

### 🔧 MCP Git инструменты
- Получение текущей git-ветки
- Список файлов в проекте с фильтрацией по расширениям
- Древовидная структура проекта
- Просмотр незакоммиченных изменений
- Чтение README файлов

### 📁 MCP инструменты для работы с файлами
- **Чтение файлов** - чтение содержимого файлов проекта
- **Поиск по файлам** - поиск текста в файлах проекта
- **Анализ импортов** - поиск всех мест, где импортируется указанный модуль
- **Список файлов** - получение списка файлов по шаблону
- **Анализ файлов** - анализ структуры файла (импорты, функции, классы)
- **Создание файлов** - создание новых файлов с содержимым
- **Изменение файлов** - добавление содержимого в существующие файлы

### 🔒 Система безопасности
- Проверка путей на нахождение внутри проекта
- Автоматическое создание бэкапов при изменении файлов
- Ограничение глубины поиска файлов
- Исключение системных директорий (.git, __pycache__, venv и др.)

### 💬 Команды

**Команды разработчика:**
- `/help` - список всех команд и примеры вопросов
- `/branch` - текущая git-ветка
- `/files` - файлы проекта
- `/structure` - структура проекта
- `/diff` - незакоммиченные изменения
- `/readme` - содержимое README

**📁 Команды работы с файлами:**
- `/read <file>` - прочитать содержимое файла
- `/find "текст"` - поиск по файлам проекта
- `/imports <module>` - найти все импорты модуля
- `/files <pattern>` - список файлов по шаблону
- `/analyze <file>` - анализ файла (структура, функции)
- `/create <file>` - создать новый файл
- `/append <file> "text"` - добавить текст в файл

**👥 Команды поддержки пользователей:**
- `/support [вопрос]` - чат с поддержкой пользователей
- `/faq` - просмотр FAQ (часто задаваемых вопросов)
- `/users` - управление пользователями
- `/tickets` - управление тикетами поддержки
- `/stats` - статистика поддержки

## 🛠️ Установка и запуск

### Требования
- Docker и Docker Compose
- Ollama (локально или в контейнере)

### Запуск
```bash
docker compose up --build
```

Приложение будет доступно по адресу: http://localhost:8000

## 🌐 API эндпоинты

### Основные
- `GET /health` - статус приложения
- `POST /api/chat` - чат с ассистентом (с поддержкой команд)
- `GET /api/rag/search` - поиск по документации

### MCP Git API
- `GET /api/git/branch` - текущая ветка
- `GET /api/git/files` - список файлов
- `GET /api/git/structure` - структура проекта
- `GET /api/git/diff` - изменения в репозитории
- `GET /api/docs/readme` - содержимое README

## 💡 Примеры использования

1. Откройте http://localhost:8000

   - `/help` - получить справку

## 🔒 Система безопасности

### Проверка путей
- Все операции с файлами проверяются на нахождение внутри проекта
- Запрещены пути, выходящие за пределы проекта (`../`, абсолютные пути вне проекта)
- Исключены системные директории (`.git`, `__pycache__`, `venv`, `.venv`, `.file_mcp_backups`)

### Бэкапы
- При изменении существующих файлов автоматически создаются бэкапы
- Бэкапы сохраняются в директории `.file_mcp_backups/`
- Формат имен бэкапов: `filename.backup_YYYYMMDD_HHMMSS`

### Ограничения
- Максимальная глубина поиска файлов: 5 уровней
- Максимальное количество результатов поиска: 20
- Максимальное количество строк при чтении: 100 (по умолчанию)

## 🔄 Архитектура

Приложение построено на:
- **FastAPI** - веб-фреймворк
- **Ollama** - локальные LLM модели
- **FAISS** - векторный поиск (для существующего RAG)
- **Git MCP** - интеграция с Git через subprocess
- **File MCP** - инструменты для работы с файлами проекта
- **Простой поиск по ключевым словам** - для документации проекта

## 🔌 Расширение функциональности

### Добавление новых команд:
1. Добавить обработку в `app/ollama_client.py::_handle_command()`
2. Добавить функцию в `app/mcp_tools.py` или `app/file_mcp.py`
3. Обновить команду `/help`

### Добавление документации:
1. Добавить файлы в папку `docs/`
2. Они будут автоматически загружены при перезапуске

### Добавление новых инструментов работы с файлами:
1. Добавить метод в класс `FileMCP` в `app/file_mcp.py`
2. Добавить обработку команды в `app/ollama_client.py`
3. Обновить документацию
