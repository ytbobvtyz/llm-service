# 🚚 Логистический агент для анализа ограничений на просушку дорог

AI-агент, который помогает сотрудникам отдела логистики планировать рейсы по РФ в период паводков. Агент анализирует маршруты, определяет регионы следования, находит в RAG-памяти постановления об ограничениях на просушку дорог и выдаёт рекомендации с альтернативными маршрутами.

## 📋 Возможности

- **Построение маршрутов** через Яндекс Router API с альтернативными вариантами
- **Определение регионов** следования по полилинии маршрута
- **Поиск ограничений** в базе документов (PDF, DOCX, TXT) по регионам
- **Анализ ограничений** на нагрузку на ось, периоды действия и другие параметры
- **Рекомендации** по альтернативным маршрутам при наличии ограничений
- **Веб-интерфейс** с чатом и отладочной панелью в реальном времени
- **Автоматическая индексация** документов с извлечением метаданных
- **Контейнеризация** с Docker для легкого развертывания

## 🔧 Локальная установка (без Docker)

### 1. Установка зависимостей

```bash
# Установка зависимостей Python
pip install -r requirements.txt

# Или установка основных пакетов
pip install fastapi uvicorn chromadb sentence-transformers langchain langchain-community ollama
```

### 2. Настройка окружения

Создайте файл `.env` в корне проекта:

```env
YANDEX_MAPS_API_KEY=ваш_ключ_яндекс_карт
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama3.2:3b
CHROMA_DB_PATH=./chroma_db
RESOLUTIONS_PATH=./resolutions
```

**Для получения Яндекс API ключа:**
1. Перейдите на [Яндекс.Облако](https://cloud.yandex.ru/)
2. Создайте сервисный аккаунт
3. Получите API ключ для "Геокодер" и "Router API"

### 3. Подготовка документов

Поместите документы с ограничениями в папку `resolutions/`. Поддерживаемые форматы: TXT, PDF, DOCX.

**Формат имен файлов:**
```
{Регион}_{тип_документа}_{номер}.{расширение}
```

**Примеры:**
- `Пермский край_постановление_123.pdf`
- `Татарстан_распоряжение_45.docx`
- `Московская область_приказ_78.txt`

Структура по годам:
```
resolutions/
├── 2026/
│   ├── Пермский край_постановление_123.txt
│   └── Татарстан_постановление_45.txt
└── 2025/
    └── ...
```

### 4. Индексация документов

```bash
# Запуск индексации
python -m indexing.indexer

# Или с подробным выводом
python -m indexing.indexer --verbose
```

### 5. Запуск сервера

```bash
# Запуск сервера разработки
python app/main.py

# Или через uvicorn напрямую
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

Сервер будет доступен по адресу: http://localhost:8000

## 🛠️ Технологический стек

- **Backend:** FastAPI + Uvicorn
- **AI-агент:** LangChain с кастомными инструментами
- **Локальная LLM:** Ollama (llama3.2, mistral и др.)
- **Векторная БД:** ChromaDB
- **Индексация документов:** PyMuPDF (PDF), python-docx (DOCX), chardet (TXT)
- **Геоданные:** Яндекс Router API + Геокодер
- **Фронтенд:** HTML/CSS/JS с SSE для реального времени
- **Контейнеризация:** Docker + Docker Compose
- **Конфигурация:** Pydantic Settings + .env

## 📁 Структура проекта

### Основные модули

1. **`indexing/indexer.py`** – индексатор документов
   - Рекурсивный обход папки `resolutions/`
   - Извлечение текста из PDF/DOCX/TXT
   - Извлечение метаданных из имен файлов
   - Чанкинг текста (1000 токенов с перехлестом 100)
   - Сохранение в ChromaDB с метаданными

2. **`agent/tools/`** – инструменты агента
   - `yandex_routes.py` – построение маршрутов через Яндекс API
   - `yandex_geocoder.py` – геокодирование и определение регионов
   - `rag_search.py` – поиск ограничений в векторной БД

3. **`agent/core.py`** – ядро агента
   - Оркестрация вызовов инструментов
   - Извлечение сущностей из запросов
   - Формирование структурированных ответов
   - Обработка ошибок и таймаутов

4. **`app/api/routes.py`** – API слой
   - RESTful эндпоинты
   - SSE для отладочной информации
   - Rate limiting
   - Валидация запросов

## ⚠️ Ограничения и известные проблемы

1. **Зависимость от внешних API:**
   - Яндекс Router API имеет квоты на запросы
   - При недоступности API маршруты не строятся

2. **Качество поиска ограничений** зависит от:
   - Качества документов в `resolutions/`
   - Формата имен файлов
   - Размера чанков при индексации

3. **Производительность:**
   - Обратное геокодирование выполняется последовательно
   - Для больших полилиний может требоваться время

4. **Требования к Ollama:**
   - Должна быть установлена и запущена локально
   - Рекомендуемая модель: llama3.2:3b или лучше
