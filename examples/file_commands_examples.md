# Примеры использования команд работы с файлами

## Сценарии использования

### Сценарий 1: Исследование кодовой базы

**Задача:** Понять, как работает система обработки команд в проекте.

**Команды:**
```
/find "class OllamaClient"
```
→ Находит файл `app/ollama_client.py` с определением класса

```
/read app/ollama_client.py
```
→ Читает содержимое файла, показывает структуру

```
/analyze app/ollama_client.py
```
→ Анализирует файл: показывает импорты, функции, классы

```
/imports httpx
```
→ Находит все места, где используется HTTP клиент

### Сценарий 2: Создание документации

**Задача:** Создать документацию для новой функции.

**Команды:**
```
/create docs/NEW_FEATURE.md "# Новая функция: Работа с файлами\n\n## Описание\n\nАссистент теперь может работать с файлами проекта через MCP инструменты.\n\n## Команды\n\n- `/read` - чтение файлов\n- `/find` - поиск по файлам\n- `/imports` - поиск импортов\n\n## Примеры\n\nСм. examples/file_commands_examples.md"
```

```
/append README.md "\n## Работа с файлами\n\nАссистент поддерживает команды для работы с файлами проекта. Подробнее в docs/FILE_COMMANDS.md"
```

### Сценарий 3: Поиск проблем в коде

**Задача:** Найти все места, где используется устаревший API.

**Команды:**
```
/find "deprecated_function"
```
→ Находит все использования устаревшей функции

```
/files *.py
```
→ Получает список всех Python файлов для ручной проверки

```
/imports old_module
```
→ Находит импорты устаревшего модуля

### Сценарий 4: Анализ структуры проекта

**Задача:** Понять архитектуру проекта.

**Команды:**
```
/structure
```
→ Показывает древовидную структуру проекта

```
/files app/*.py
```
→ Показывает все Python файлы в папке app

```
/analyze app/main.py
```
→ Анализирует главный файл приложения

```
/analyze app/api.py
```
→ Анализирует API эндпоинты

## Практические примеры диалогов

### Пример 1: Новый разработчик изучает проект
```
Разработчик: /help
Ассистент: Показывает список всех команд

Разработчик: /structure
Ассистент: Показывает структуру проекта

Разработчик: /read app/main.py
Ассистент: Показывает код главного файла

Разработчик: /find "def chat_with_commands"
Ассистент: Находит функцию обработки команд
```

### Пример 2: Создание отчета
```
Разработчик: /create report.md "# Отчет по проекту\n\n## Файлы проекта"
Ассистент: Создает файл отчета

Разработчик: /append report.md "\n## Python файлы:\n"
Ассистент: Добавляет заголовок

Разработчик: /files *.py
Ассистент: Показывает список Python файлов

Разработчик: /append report.md "\n- app/main.py\n- app/api.py\n- app/ollama_client.py"
Ассистент: Добавляет список файлов в отчет
```

### Пример 3: Отладка проблемы
```
Разработчик: /find "error_message"
Ассистент: Находит все места с сообщением об ошибке

Разработчик: /read app/file_mcp.py
Ассистент: Показывает код инструментов работы с файлами

Разработчик: /analyze app/file_mcp.py
Ассистент: Анализирует структуру файла

Разработчик: /imports os
Ассистент: Показывает все импорты модуля OS
```

```
# 1. Получить структуру
/structure

# 2. Найти все классы
/find "class "

# 4. Проанализировать основные файлы
/analyze app/main.py

/analyze app/ollama_client.py
```

```

# 3. Добавить информацию о классах
/append docs/CODEBASE_OVERVIEW.md "## Основные классы\n\n"

```

## Советы по эффективному использованию

### Для поиска кода
1. Используйте точные фразы в кавычках
2. Комбинируйте `/find` с `/read` для детального изучения
3. Используйте `/imports` для понимания зависимостей

### Для анализа проекта
1. Начните с `/structure` для общего понимания
2. Используйте `/files` для навигации
3. Применяйте `/analyze` для детального изучения файлов

### Для создания документации
1. Используйте `/create` для новых файлов
2. Применяйте `/append` для добавления контента
3. Комбинируйте с другими командами для сбора информации

## Ограничения и обходные пути

### Ограничение: Максимальное количество строк
**Проблема:** `/read` показывает только 100 строк по умолчанию
**Решение:** Используйте несколько команд для чтения разных частей файла

### Ограничение: Глубина поиска
**Проблема:** `/files` ищет только на глубине 3 уровня
**Решение:** Используйте конкретные пути (например, `app/**/*.py`)

### Ограничение: Количество результатов
**Проблема:** `/find` показывает только 10 файлов
**Решение:** Уточняйте запрос для получения более релевантных результатов
